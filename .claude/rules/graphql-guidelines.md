# GraphQL Query Rules

These rules apply when adding or modifying `.graphql` files or working with
GraphQL domain packages under `pkg/polaris/graphql/`.

## Query Structure

- Operation name MUST be `RubrikPolarisSDKRequest` (both queries and mutations)
- Result MUST be aliased to `result:`
- Input fields MUST be extrapolated as individual `$variables` — never pass
  complex input types directly

```graphql
# Correct
mutation RubrikPolarisSDKRequest($id: String!, $name: String!) {
  result: updateThing(input: { id: $id, name: $name }) { id }
}

# Wrong — complex input type
mutation RubrikPolarisSDKRequest($input: UpdateThingInput!) {
  result: updateThing(input: $input) { id }
}
```

## File Conventions

- Place `.graphql` files in `pkg/polaris/graphql/<domain>/queries/`
- Use `snake_case` for file names (generator converts to camelCase variables)
- GraphQL files use the API's field capitalization (e.g., `Id` not `ID`)

## Generation

- Run `go generate ./...` after any `.graphql` change
- NEVER manually edit `queries.go` — it has the header
  `// Code generated by queries_gen.go DO NOT EDIT.`
- Commit both the `.graphql` source and the regenerated `queries.go`
