# CLAUDE.md

## Repository Overview

**rubrik-polaris-sdk-for-go** is the official Go SDK for Rubrik Security Cloud
(RSC). It wraps the RSC GraphQL API into idiomatic Go types and methods,
consumed primarily by the `terraform-provider-polaris` Terraform provider.

- **Module:** `github.com/rubrikinc/rubrik-polaris-sdk-for-go`
- **Go version:** 1.24+
- **License:** MIT
- **Main branch:** `main`

## Key Directories

| Path | Description |
|------|-------------|
| `pkg/polaris/graphql/` | Domain packages — each subdirectory wraps one RSC API area |
| `pkg/polaris/graphql/*/queries/` | `.graphql` files for each domain (source of truth for queries) |
| `pkg/polaris/graphql/regions/` | Strongly-typed region packages (`aws`, `azure`, `gcp`) |
| `pkg/polaris/graphql/core/` | Shared types, enums, permissions, feature flags |
| `pkg/polaris/` | High-level API wrappers (aws/, azure/, gcp/, access/, etc.) |
| `internal/` | Test helpers: `handler/` (mock HTTP), `assert/`, `testsetup/` |
| `cmd/` | CLI tools: `schema` (introspection), `testenv` |
| `examples/` | 30+ example programs |

## Build & Test

```bash
go build ./...           # Build all packages
go generate ./...        # Regenerate queries.go from .graphql files
go fmt ./...             # Format
go vet ./...             # Lint
go test ./...            # Run unit tests
```

Integration tests require `TEST_INTEGRATION=1` and RSC credentials in
`~/.rubrik/polaris-accounts.json`.

## GraphQL Code Generation

This is the most common workflow. Each domain package under
`pkg/polaris/graphql/` has a `queries/` subdirectory containing `.graphql`
files. A custom generator (`queries_gen.go`) compiles these into `queries.go`.

**Workflow:** Edit `.graphql` files -> `go generate ./...` -> commit both.

**Rules:**
- NEVER manually edit `queries.go` (header: `// Code generated by queries_gen.go DO NOT EDIT.`)
- ALL queries/mutations MUST use `RubrikPolarisSDKRequest` as the operation name
- ALL results MUST be aliased to `result:`
- Extrapolate input fields as individual parameters — don't pass complex input types
- File names use `snake_case.graphql` (generator converts to camelCase Go variables)

```graphql
mutation RubrikPolarisSDKRequest($cloudAccountId: String!, $name: String!) {
  result: updateAwsCloudAccount(input: {
    cloudAccountId: $cloudAccountId,
    name: $name,
  }) {
    id
  }
}
```

Use the `/rx-queries` skill to run generation for modified domains.

## Region Types

ALWAYS use typed regions, never strings:

| Provider | Type | Package |
|----------|------|---------|
| AWS | `aws.Region` | `.../graphql/regions/aws` |
| Azure | `azure.Region` | `.../graphql/regions/azure` |
| GCP | `gcp.Region` | `.../graphql/regions/gcp` |

Parse with `RegionFromName()`, `RegionFromAny()`. Convert with
`.ToRegionEnum()`. Validate against `RegionUnknown`.

## Sensitive Data

Use `secret.String` from `pkg/polaris/graphql/core/secret` for credentials and
tokens — never bare `string`. This prevents accidental logging of secrets.

## Code Style

- MIT license header on all source files (copyright Rubrik, Inc.)
- Doc comments on all exported functions and types
- Single Responsibility Principle for functions
- Dependency injection for testability; prefer fakes over mocks in tests
- CamelCase test function names
- Follow patterns in adjacent code — consistency over novelty
- Refer to [Effective Go](https://go.dev/doc/effective_go) for general conventions

## Commit & PR Conventions

- Commit messages: imperative mood, no trailing period (e.g., `Add BAAS_BASIC permission group`)
- PR numbers appended automatically by squash-merge (`#NNN`)
- PRs require: linked issue, test description, CHANGELOG update
- Versioning: SemVer (`vMAJOR.MINOR.PATCH`)

## Pre-Commit Checklist

1. `go fmt ./...`
2. `go vet ./...`
3. `go generate ./...` (if any `.graphql` files changed)
4. `go test ./...`
5. No manual edits to generated files
6. Region types used (not strings)
7. `secret.String` used for sensitive fields
